## Surface Anti-Drift Acceptance

This proposal is not a UI implementation proposal yet. If a later child change adds UI surfaces, it must create or update `ui-design.md` and `ui-acceptance.md` before browser implementation.

### forbidden_actions

1. Show submit/cancel/replace/modify controls from ADR-0005 observation state.
2. Parse raw broker callback payloads in the browser as account/order truth.
3. Display connected observation as live readiness, broker tradability, capital approval or command authorization.
4. Add broker-specific Account Console UI truth branches for CTP, IB, stock, CQG or TT.
5. Render execution reports only as cards, free text or raw payload dumps without a normalized table.

### forbidden_claims

1. ADR-0005 proves live trading readiness.
2. Observation session connectivity permits order mutation.
3. Screenshot evidence proves broker/account/order truth.
4. Raw broker payload is canonical order report truth.
5. Memory-only callback state is sufficient for post-session review.
6. A startup callback snapshot proves complete execution history without durable-store replay evidence.

## Implementation/browser evidence

Implementation/browser evidence is complete for the ADR-0005/P019 foundation closeout. Real order/fill report-row parity remains a residual runtime blocker because the current authorized same-slice `reqExecutions` query returned zero rows.
