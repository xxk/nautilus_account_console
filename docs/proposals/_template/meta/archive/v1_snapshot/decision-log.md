## 使用约束

1. 一条决策只记录一个稳定结论，不把多个无关动作揉成一条长日志。
2. `决策 / 原因 / 回写动作 / 明确不做` 四段尽量保持完整，便于后续 proposal 复盘。
3. `decision-log.md` 记录的是“为什么这样定、做了什么回写、哪些事情明确不做”，不是 phase 状态板，也不是 child change 执行台账。
