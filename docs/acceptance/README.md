## Usage Rules

1. Use implementation UI acceptance before accepting any frontend route or panel.
2. Each proposal should add its own panel-level `ui-acceptance.md` when it owns a concrete UI slice.
3. Acceptance evidence must distinguish source checks, browser checks and typed blockers.
4. Every UI proposal/change must complete the anti-drift checklist before code starts.
5. Every UI proposal/change that adds, implements or promotes a route must update the route coverage matrix.
6. Every UI proposal/change closeout must open the touched UI route after the final implementation change and record browser/render evidence or a typed blocker.
